## Understanding Copilot availability: Geography, languages, and localization

One of the most common sources of confusion when reviewing Copilot feature availability is understanding the relationship between three distinct but interconnected concepts: **geographic availability**, **supported languages**, and **environment localization**. These dimensions work independently of each other, creating a matrix of availability that can seem complex at first glance. This article clarifies these concepts to help you better understand when and how Copilot features become available to your organization.

### Why Microsoft doesn't release everywhere simultaneously

Microsoft takes a measured, phased approach to releasing Copilot features across geographies and languages. This strategy isn't arbitrary—it's driven by several critical factors:

**Regulatory and compliance requirements**: Different countries and regions have varying data residency, privacy, and AI governance regulations. Before launching Copilot features in a new geography, Microsoft must ensure full compliance with local laws, including regulations around data processing, AI transparency, and customer consent. For example, features must meet GDPR requirements in Europe, which may differ from regulations in other regions.

**Azure infrastructure dependencies**: Copilot features rely on specific Azure AI services and infrastructure. These services must be deployed and operational in Azure data centers within a given geography before features can be made available to customers in that region. The "Geo" column in the table above indicates which Azure geographies currently support each feature's underlying infrastructure.

**Quality and performance validation**: Each new geography requires extensive testing to ensure that Copilot features perform reliably under local network conditions, latency constraints, and regional usage patterns. Microsoft validates performance benchmarks before expanding availability.

**Language model quality**: For features that support multiple languages, Microsoft must train and validate AI models for each language to ensure high-quality outputs. This testing process takes time and resources, which is why features often launch in English first and expand to additional languages over subsequent releases.

This phased approach ensures that when a feature becomes available in your geography and language, it delivers a reliable, compliant, and high-quality experience.

### The three dimensions of availability

To understand Copilot feature availability, you need to consider three independent dimensions:

#### 1. Geographic availability (Azure Geo)

The **Geo** column in the table indicates the Azure geographic regions where each feature's underlying AI services are deployed and operational. This is about data center infrastructure and where the AI processing occurs.

**Important distinction**: Geographic availability is NOT the same as country availability. For example:
- "Europe" in the Geo column means the feature is supported in Azure's European data centers
- This doesn't automatically mean every European country can access the feature—country availability also depends on licensing, regulatory approval, and Business Central's own market presence
- A feature available in "Asia Pacific" geography might still be restricted in certain countries within that region due to regulatory constraints

**When "Not specified" appears**: Some features like "Autofill fields" and "Suggest item substitutions" show "Not specified" in the Geo column. This typically means one of two things:
- The feature may have global availability without geography-specific restrictions
- Geographic rollout details are still being finalized

#### 2. Supported languages (tested and validated)

The **Supported Languages** column lists languages in which Microsoft has explicitly tested and validated each feature. These are languages where:
- The feature has undergone quality assurance testing
- AI model outputs have been evaluated for accuracy, appropriateness, and cultural relevance
- Documentation and error messages are available
- Microsoft provides official support if issues arise

**Notice the variation**: Different features support different language sets. For example:
- "Analysis assist" supports 22 languages including Chinese, Greek, and Thai
- "Payables Agent" currently supports only English variants
- "Sales Order Agent" supports 14 languages including Icelandic but not Chinese

This variation exists because each feature uses different underlying AI models and technologies, each with its own language training requirements and validation timeline.

#### 3. Environment localization (user interface language)

Environment localization refers to the language your Business Central environment displays in—the language of menus, buttons, labels, and other interface elements. This is controlled by your user settings and is independent of both geography and Copilot feature language support.

**Critical distinction**: Your environment's display language does NOT determine whether you can use a Copilot feature. The determining factors are:
- Whether your organization's environment is hosted in a supported Azure geography
- Whether you're interacting with the feature in one of its supported languages

### Supported vs. unlocked languages: Understanding the difference

This is perhaps the most misunderstood aspect of Copilot language availability.

**Supported languages** (listed in the table):
- Officially tested and validated by Microsoft
- Quality metrics have been established and met
- Full support is available if you encounter issues
- Documentation reflects these languages
- Expected to deliver consistent, high-quality results

**Unlocked (untested) languages**:
- Features may technically function in languages not listed in the table
- These languages haven't undergone rigorous testing by Microsoft
- Quality, accuracy, and appropriateness of AI-generated content are NOT guaranteed
- No official support is provided for issues specific to these languages
- Results may vary significantly based on the AI model's general language capabilities

**Why features work in unlocked languages**: The underlying AI models (like Azure OpenAI Service) often have broad multilingual capabilities. When you use a Copilot feature in an untested language, the AI may still generate responses, but Microsoft hasn't validated that these responses meet quality standards.

**The risk**: Using features in unlocked languages might produce:
- Inaccurate or inappropriate translations
- Culturally insensitive content
- Mixing of languages in outputs
- Lower accuracy in understanding prompts or generating relevant suggestions

**Our recommendation**: For production environments and business-critical processes, use Copilot features only in their officially supported languages. You may experiment with unlocked languages in test environments, but understand that results aren't guaranteed and support is limited.

### Common scenarios that cause confusion

#### Scenario 1: "The feature is available in my geography, but not in my language"

**Example**: You're in Brazil (covered under "Brazil" in the Geo column), but your primary business language is Portuguese. Looking at "Payables Agent," you see:
- Geo: Includes Australia, Canada, Europe, United Kingdom, United States (Brazil is NOT listed)
- Supported Languages: English variants only

**Explanation**: Even though Brazil isn't currently in the supported geography list for Payables Agent, this demonstrates the language limitation. If Brazil were added to the Geo list, the feature would still only support English. You could use it by switching your interaction language to English, but the feature wouldn't support Portuguese interactions.

#### Scenario 2: "My environment is in French, but the feature only supports English"

**Example**: Your Business Central environment is localized in French (fr-FR), and you want to use "Suggest item substitutions."
- Geo: Not specified (potentially available)
- Supported Languages: English only

**Explanation**: You can still use this feature! Change your interaction language to English when working with this specific Copilot feature. Your environment's overall localization doesn't prevent access—you simply need to interact with the feature in a supported language.

#### Scenario 3: "The geography list includes 'Europe,' but the feature doesn't work in my European country"

**Example**: You're in Portugal, and "Europe" is listed in the Geo column, but you can't access the feature.

**Explanation**: The Geo column indicates Azure infrastructure availability, but individual country access also depends on:
- Whether Business Central is licensed and available in your specific country
- Local regulatory approvals for AI features
- Your organization's specific license and settings

Contact your Microsoft account team or partner to understand country-specific availability.

### How to determine if a feature is available to you

Follow this checklist:

1. **Check geography**: Is your Azure region listed in the Geo column? (Your admin can confirm which Azure region hosts your environment)

2. **Check language support**: Can you interact with the feature in one of the supported languages listed?

3. **Check licensing**: Does your organization have the required Business Central license that includes Copilot features?

4. **Check admin settings**: Have Copilot features been enabled by your administrator?

5. **Check country availability**: Is Business Central Copilot approved and available in your specific country? Refer to the [Copilot international availability report](https://aka.ms/bapcopilot-intl-report-external) for the latest country-level details.

All five conditions must be met for a feature to be accessible.

### Looking ahead

Microsoft continually expands Copilot availability across new geographies and languages. The availability shown in the table above represents a snapshot as of February 2026. 

**Expect expansion**: Features that currently support only English often expand to additional languages in subsequent releases. Features available in limited geographies typically expand to new regions as infrastructure and compliance requirements are met.

**Stay informed**: Bookmark the [Copilot international availability report](https://aka.ms/bapcopilot-intl-report-external) for the most current information about geographic and language availability. Release plans and product announcements provide advance notice of upcoming expansions.

### Key takeaways

- **Geographic availability** (Geo) indicates where Azure infrastructure supports the feature—it's NOT the same as country-by-country availability
- **Supported languages** are tested and validated—using features in other languages is unsupported and may produce unreliable results
- **Environment localization** (UI language) is independent of feature availability—you can use a feature in a supported language regardless of your environment's display language
- These three dimensions work independently, creating a complex availability matrix
- Microsoft's phased rollout approach prioritizes compliance, quality, and performance over speed
- Always check all five conditions (geography, language, licensing, admin settings, and country approval) to determine feature availability

Understanding these distinctions will help you set accurate expectations, plan rollouts effectively, and avoid confusion when reviewing release plans and availability reports.
